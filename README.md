# testDelivery

Тестовое задание: Экран главной страницы (React Native)

## Стек технологий

- **React Native CLI** (без Expo)
- **TypeScript**
- **TanStack Query (React Query) v5** — управление состоянием и кеширование
- **MMKV** — быстрое локальное хранилище для персистентного кеша
- **React Native Reanimated & Gesture Handler** — анимации
- **Gorhom Bottom Sheet** — реализация выпадающего списка ресторанов

## Архитектура

Проект организован по принципам **FSD** (упрощенно):

- `src/app`: Провайдеры и инициализация приложения.
- `src/screens`: Экраны (в данном случае `Home`).
- `src/entities`: Бизнес-сущности (`ads`, `vendor`) с их API и типами.
- `src/shared`: Переиспользуемые модули (http-клиент, конфиги, хранилище).

## Особенности реализации

- **Кеширование**: Данные сохраняются в MMKV и доступны офлайн. При перезапуске приложение сначала показывает данные из кеша.
- **Обновление**: Использована стратегия `stale-while-revalidate`. При фокусе на приложении (`focusManager`) данные обновляются в фоне.
- **Производительность**: Для списка ресторанов используется `BottomSheetFlatList` для эффективной виртуализации.

## Как запустить

1. **Установка зависимостей**:

   ```sh
   bun install
   ```

   (или `npm install`)

2. **Android**:
   Убедитесь, что эмулятор запущен, затем:

   ```sh
   bun android
   ```

3. **iOS** (только macOS):
   ```sh
   cd ios && bundle install && bundle exec pod install && cd ..
   bun ios
   ```
